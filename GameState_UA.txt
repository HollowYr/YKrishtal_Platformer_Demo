[Що є у грі]
 = Анімації = 
   
   Персонаж: 
   біг, Idle, комбо удари з мечем, стрибок,
   падіння, присідання та переміщення присядки, cмерть.
   
   Slime:
   пересування, idle, атака, hurt та смерть.
   
   Освітлення:
   Великий та римський факели, багаття в головному меню.
   
   Перехід між сценами:
   Fade in/out.
   
   UI:
   Анімація кнопки "Retry" при смерті, пофрагментне поповнення сердець.
 
 
 = Звуки =

   Персонаж:
   Атака з мечем (1-3), смерть, отримання шкоди. 
   
   Slime:
   Отримання шкоди, смерть.
   
   Музика:
   Одна тема в головному меню,
   яка продовжується в наступних рівнях та зациклена

 = Скріпти =

   Enemy Behaviour: 
    Slime Move: скрипт патрулювання.
    Slime Idle: скрипт переведення в режим патрулювання або атаки.

   Player Control:
    BetterJump: 
    Дозволяє змінювати висоту стрибка за допомогою затискання кнопки. 
    Довше тримаєш - вище стрибаєш.

    CharacterController2D:
    контроллер, що відповідає за рух персонажа. (переміщення, стрибки, присідання)
    
    PlayerCombo: 
    Дозволяє провести комбо удари, але лише після певного часу як закінчилась анімація
    для уникнення спаму кнопки атаки. Також якщо пройде забагато часу - наступна анімація 
    атаки буде скинутою до стандартної.

    PlayerMovement: 
    Зчитує Input з кнопок та передає у CharacterController2D. Також керує анімаціями переміщення.
    
    PlayerScript:
    Контейнер для всіх скріптів пов'язаних з гравцем аби імплементувати лише один скрипт
    маючи при цьому доступ до всіх необхідних.
   
   UI:
    AntiSpamBtn: 
    Виконує UnityEvent вдруге лише коли пройде певний час. (Для уникнення спаму кнопок)

    CanvasSwitcher:
    Робить неактивними всі групи UI всередині SafeArea та активує задану з параметрів.

    HeartHealthSystem: Логічна частина системи здоров'я персонажа. (Віднімає, додає здоров'я)
    HeartsHealthVisual: Візуальна частина системи здоров'я персонажа. (Змінює стан UI Сердець)

    OrientationSetter: 
    Встановлює орієнтацію екрану. Портретну (Фіксовану та не фіксовану),
    пейзажну(Фіксовану та не фіксовану), з вільним поворотом екрану.    

    Parallax Background:
    створює ефект паралаксу з тлом переміщаючи його коли камера пройде частину тла.
   
    SafeAreaLimiter: 
    Змінює розміри RectTransform елемента на якому доданий так, аби вони дорівнювали 
    розмірам SafeArea екрану мобільного пристрою.

    TextWriter:
    Екземпляр текстового вікна в якому буде з'являтися текст з ефектом друкарської машинки.
    
    UI_Assistant:
    Контроллер діалогових вікон. Зчитує з файлу текст та посилає на екземпляри TextWriter.
    
   Решта:
    AudioManager:
    Містить в собі всі аудіодоріжки та метод для відтворення.
    
    CameraShake: 
    Надає доступ до налаштування Basic Multichannel Perlin Noise для виклику ефекту
    струшування камери з параметрами сили та часу струсу.

    Dissolve:
    Керує параметром Fade в однойменному матеріалі створюючи ефект розчинення об'єкта.
    Коли Fade = 0, себто об'єкт не видно - знищує об'єкт.
    
    EnemyController: 
    Основний скріпт для ворогів. З нього викликаються функції MoveTo, Attack, TakeDamage, Die.
    
    GameTime:
    Керування ігровим часом. Якщо гра не на паузі - видає звичайний deltaTime i fixedDeltaTime,
    інакше - 0. Корисно коли потрібно поставити на паузу не всі об'єкти, як це робить TimeScale.
    
    LevelLoader:
    Асинхронно завантажує сцену з номером з параметрів та запускає Fade in анімацію.
   
    OnTriggerEnter:
    Викликає UnityEvent коли активується OnTriggerEnter2D живим гравцем.
    
    SetObjectActive:
    Активує об'єкт. Використовується для анімації появи кнопок керування.
  
    Spikes:
    Відкидає гравця в протилежному напрямку завдяки AddForce та наносить йому шкоду.
